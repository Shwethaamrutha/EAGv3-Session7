**Python asyncio Best Practices (Common Advice Across Sources)**

1. **Always use `asyncio.run()` as your entry point** - Properly starts and tears down the event loop; simply defining `async` functions and calling them without `asyncio.run()` does nothing useful.

2. **Always await coroutines** - Forgetting to `await` a coroutine means it is created but never executed; every coroutine call must be awaited or scheduled explicitly.

3. **Avoid blocking the event loop** - Do not use blocking calls (e.g., `time.sleep`, long-running loops) inside async functions, as they halt the entire event loop and eliminate concurrency benefits; use `asyncio.sleep` and schedule work as tasks instead.

4. **Use tasks to run coroutines concurrently** - Schedule independent coroutines as tasks (via `asyncio.create_task`) rather than awaiting them sequentially, which would behave synchronously.

5. **Handle cancellation and resource cleanup** - Catch `CancelledError` in long-running tasks and use async context managers (`async with`) to ensure proper resource management.
